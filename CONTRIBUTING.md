# Contributing Guide

## How to Add Knowledge Documents

1. Create a new markdown file in the appropriate category folder
2. Follow the naming convention: `category-topic-version.md`
3. Include the required metadata header
4. Submit a pull request for review

## Document Format

All documents must include:

```yaml
---
title: Document Title
category: business|creative|technical|community
tags: [tag1, tag2, tag3]
version: 1.0
author: Author Name
date: YYYY-MM-DD
---
```

## Categories

- **business** - Business strategy, entrepreneurship, marketing
- **creative** - Design principles, creative tools, artistic techniques
- **technical** - Programming, APIs, system architecture
- **community** - Social impact, community guidelines, support resources

## Review Process

1. Automated validation checks
2. Technical review for accuracy
3. Editorial review for clarity
4. Final approval by knowledge team
